# LSI RAID Monitor 中文使用说明

## 项目简介

本项目用于监控 LSI / Broadcom MegaRAID 阵列卡和硬盘健康状态，支持定时采集硬盘温度、错误计数、SMART 信息、虚拟磁盘状态、控制器状态、BBU / CacheVault 状态、Patrol Read 状态和系统负载，并通过邮件发送 HTML 巡检报告。

本打包版本额外加入了 `install_and_start.sh`，用于一键安装、配置、启动、查看状态、立即发送测试报告和卸载。

## 目录结构

```text
lsi-raid-monitor/
├── install_and_start.sh      # 一键安装启动脚本
├── 中文使用说明.md           # 中文说明文档
├── lsi_collectd.py           # 数据采集脚本
├── lsi_report.py             # 邮件报告脚本
├── lsi_send_now.sh           # 原项目即时发送脚本
├── setup_cron.sh             # 原项目 cron 安装脚本
└── requirements.txt          # Python 依赖
```

## 快速安装

进入项目目录：

```bash
cd lsi-raid-monitor
```

执行一键安装：

```bash
sudo bash install_and_start.sh install
```

安装脚本会完成：

1. 检查项目文件是否完整
2. 安装系统依赖
3. 创建 Python 虚拟环境 `.venv`
4. 安装 Python 依赖
5. 交互式配置 SMTP 邮件参数和即时报警参数
6. 生成 `.env` 配置文件
7. 创建数据目录
8. 配置 `storcli64` 和 `smartctl` 免密 sudo
9. 写入 crontab 定时任务

## 安装时需要填写的内容

### SMTP 服务器

例如 QQ 邮箱：

```text
smtp.qq.com
```

### SMTP SSL 端口

常见为：

```text
465
```

### SMTP 用户名 / 发件邮箱

例如：

```text
yourname@qq.com
```

### SMTP 授权码或密码

大多数邮箱需要使用 SMTP 授权码，不建议使用邮箱登录密码。

### 收件邮箱

报告发送到哪个邮箱，就填写哪个邮箱。

### 报警收件邮箱

用于接收即时故障报警邮件，可填写多个，用英文逗号分隔，例如：

```text
ops@example.com,admin@example.com
```

留空则关闭即时报警。

报警触发条件：

- 控制器健康状态不是 Optimal
- 物理磁盘状态不是 Online / Hotspare / UGood / Optimal
- 磁盘温度达到或超过 `TEMP_WARN` / `TEMP_CRIT`
- 磁盘 SMART 告警被标记
- 媒体错误 / 其他错误 / 预测性故障计数大于 0
- 物理磁盘状态与上次采集相比发生变化（如 Onln → Failed）、磁盘新增或被移除
- 虚拟磁盘（VD）与上次采集相比发生变化：VD 状态迁移（如 Optl → Dgrd）、VD 新增或被移除
- SMART 关键属性 **5 / 187 / 188 / 197 / 198**（重映射扇区、不可恢复错误、命令超时、待映射扇区、离线不可校正）数值与上次采集相比发生变化

报警通过本地 `sendmail` 服务发送（默认 `/usr/sbin/sendmail`），需要服务器已安装并配置好邮件服务（如 Postfix、Exim、Nullmailer 等）。

### sendmail 路径

默认 `/usr/sbin/sendmail`。如果本地邮件服务不在此路径，可填写实际路径。

### 数据目录

默认：

```bash
/var/lib/lsi-monitor/data
```

### RAID 控制器编号

默认：

```bash
/c0
```

如果有多张 RAID 卡，可先执行：

```bash
sudo storcli64 show
```

### 每日报告时间

默认每天 10:00 发送日报。

如果要改为 08:30，可在安装交互中填写：

```text
每日报告小时: 8
每日报告分钟: 30
```

## 常用命令

### 安装并启动

```bash
sudo bash install_and_start.sh install
```

或：

```bash
sudo bash install_and_start.sh start
```

### 查看状态

```bash
bash install_and_start.sh status
```

### 立即采集并发送测试报告

```bash
sudo bash install_and_start.sh send-now
```

建议安装完成后先执行一次，确认 RAID 数据采集和 SMTP 邮件发送正常。

### 停止定时任务

```bash
sudo bash install_and_start.sh stop
```

### 卸载定时任务和 sudoers

```bash
sudo bash install_and_start.sh uninstall
```

卸载会删除 crontab 任务和 `/etc/sudoers.d/lsi-monitor`，但会保留 `.env`、`.venv` 和历史采集数据。

## 配置文件

安装后会生成：

```bash
.env
```

示例：

```bash
LSI_DATA_DIR="/var/lib/lsi-monitor/data"
STORCLI_PATH="/usr/local/bin/storcli64"
SMARTCTL_PATH="/usr/sbin/smartctl"
LSI_CONTROLLER="/c0"
LSI_USER="admin"
SMTP_HOST="smtp.qq.com"
SMTP_PORT="465"
SMTP_USER="yourname@qq.com"
SMTP_PASS="your_smtp_auth_code"
SMTP_FROM="yourname@qq.com"
SMTP_TO="yourname@qq.com"
ALERT_EMAIL_TO=""
SENDMAIL_PATH="/usr/sbin/sendmail"
TEMP_WARN="45"
TEMP_CRIT="50"
REPORT_HOUR="10"
REPORT_MINUTE="0"
WEB_PASSWORD=""
```

报警配置也可以直接编辑数据目录下的 `alert_config.json`：

```bash
/var/lib/lsi-monitor/data/alert_config.json
```

示例内容：

```json
{
  "alert_email_to": "ops@example.com,admin@example.com",
  "sendmail_path": "/usr/sbin/sendmail",
  "temp_warn": 45,
  "temp_crit": 50
}
```

注意：`.env` 包含 SMTP 授权码，不要上传到公开仓库。

## 磁盘管理

Web 界面内置磁盘管理页面，通过 `lsblk` 统一枚举服务器上所有块设备（不依赖 RAID 卡），显示每块磁盘和分区的容量、文件系统与挂载点。

支持的操作（均通过免密 `sudo` 执行，相关 sudoers 条目由 `install_and_start.sh` 自动配置）：

- 挂载 / 卸载分区上的文件系统
- 使用 `mkfs.ext4` 或 `mkfs.xfs` 格式化磁盘 / 分区
- 查看任意磁盘的 SMART 信息

注意：格式化会清空目标设备上的全部数据。界面要求二次确认，并且包含已挂载系统分区的磁盘受保护，无法从界面上格式化或卸载。

## 用户管理

Web 界面支持基于角色的用户管理，用户保存在 `$LSI_DATA_DIR/users.json`（口令以 PBKDF2 哈希存储）：

- **管理员（admin）**：拥有全部权限，包括磁盘上下线、挂载 / 卸载、格式化、采集配置、报警配置、用户管理等危险操作
- **只读用户（viewer）**：只能查看，所有写操作接口返回 403，界面上的操作按钮自动隐藏

初始化 / 迁移规则：

- 首次启动时若尚无用户且设置了 `WEB_PASSWORD`，自动迁移出管理员账号（用户名 `admin`，口令即 `WEB_PASSWORD`）。
- 若既无用户也未设置 `WEB_PASSWORD`，保持免认证，所有功能不限制（与旧版一致）。

管理员登录后可通过页面右上角的 **用户管理** 按钮新建用户、重置口令、删除用户（也可通过 `/api/users` 接口操作）。不能删除当前登录用户，也不能删除最后一个管理员。

## 安全说明

- **默认无认证。** `users.json` 中没有任何用户时，任何能访问 Web 端口的人都能使用全部功能，请仅在可信内网运行，保持 `FLASK_RUN_HOST=127.0.0.1` 或置于防火墙之后。
- **建议创建管理员账号**（或设置 `WEB_PASSWORD`，首次启动时自动迁移为 `admin` 用户）启用认证。启用后，写操作（磁盘操作、格式化、挂载、手动采集、报警配置、用户管理）仅管理员可用，只读用户仅可查看；`WEB_SECRET_KEY` 可选，用于固定 Flask 会话密钥。
- **高危磁盘操作：** 格式化在界面上需要二次确认，系统盘受保护无法格式化 / 卸载。sudoers 仅放行所需的具体命令（`storcli64`、`smartctl`、`mount`、`umount`、`mkfs.ext4`、`mkfs.xfs`、`mkdir`、`lsblk`），安装后可检查 `/etc/sudoers.d/lsi-monitor`。

## 故障排查

### 找不到 storcli64

检查：

```bash
which storcli64
sudo find / -name storcli64 2>/dev/null
```

如果路径不是 `/usr/local/bin/storcli64`，重新执行安装脚本，并填写正确路径。

### 找不到 smartctl

安装：

```bash
sudo apt-get update
sudo apt-get install -y smartmontools
```

检查：

```bash
which smartctl
```

### 收不到邮件

检查 `.env` 中的 SMTP 配置：

```bash
cat .env
```

手动发送测试报告：

```bash
sudo bash install_and_start.sh send-now
```

查看报告日志：

```bash
tail -n 100 /var/lib/lsi-monitor/report.log
```

常见原因：SMTP 授权码错误、邮箱未开启 SMTP、服务器无法访问 465 端口、发件邮箱和 SMTP 用户不一致。

### 收不到即时报警邮件

1. 确认 `.env` 中 `ALERT_EMAIL_TO` 已填写正确的收件邮箱。
2. 确认服务器本地邮件服务可用：

```bash
which sendmail
ls -l /usr/sbin/sendmail
```

3. 手动测试报警邮件发送：

```bash
cd lsi-raid-monitor
source .venv/bin/activate
set -a; source .env; set +a
ALERT_EMAIL_TO=you@example.com python3 lsi_alert.py
```

4. 查看采集日志确认是否有报警相关输出：

```bash
tail -n 100 /var/lib/lsi-monitor/collectd.log
```

5. 启动 Web 界面后，可在页面上直接配置报警参数并点击 **保存配置** / **测试报警**：

```bash
bash start_web.sh
```

然后打开 `http://127.0.0.1:5200`，在“邮件报警配置”卡片中填写收件人、sendmail 路径和温度阈值后保存。配置会写入 `$LSI_DATA_DIR/alert_config.json`，`lsi_collectd.py` 每分钟自动读取。

### 没有采集到数据

手动测试：

```bash
sudo bash install_and_start.sh send-now
```

查看采集日志：

```bash
tail -n 100 /var/lib/lsi-monitor/collectd.log
```

常见原因：`storcli64` 路径错误、控制器编号不是 `/c0`、机器没有 MegaRAID 阵列卡、sudoers 配置异常。

```bash
tar -xzf lsi-raid-monitor-onekey.tar.gz
```

收到测试邮件并且日志没有明显错误，即部署完成。
